Offsets

Anchor Offset

Sometimes, we want to move the point on the anchor element that Popeye is anchored to.

We take the same example as in the Basic Usage guide, then add some extra props.

click me
<a key="anchor1" className="button button-main" onClick={ Popeye.toggleHandler(this, "popover1") }> click me <Popeye anchorOffset={ {left: 0, bottom: -16} } opened={ this.state.popover1 } onToggle={ Popeye.toggleHandler(this, "popover1") }> <div className="dropdown"> Aye aye, Captain! </div> </Popeye> </a>

Our coordinate system is similar to html/css absolute and relative positioning. We use top, bottom, left, and right, but only one of each axis at a time. For example, we can use top and left, but never top and bottom.

Each side-value pair describes a point on the anchor relative to the side. {left: 7, bottom: 13} would have described a point fully inside the boundaries of the <a>.

As it is, the side-value pair is {left: 0, bottom: -16}. The point is therefore outside the boundaries of the <a>, exactly 16 pixels below it.

Pop Offset

... And sometimes, we want to move the point on Popeye that the anchor point is attached to.

This has the same effect as the example above illustrating anchor offsetting, except for one difference: the origin of the animation. In the previous example, the Popeye appeared to have come a point on its upper-left. In this example, the origin appeared to have come from the lower left of the anchor.

click me 2
<a key="anchor3" className="button button-main" onClick={ Popeye.toggleHandler(this, "popover3") }> click me 3 <Popeye anchorOffset={ (anchor, pop) => { const dims = anchor.getBoundingClientRect(); return { top: dims.height / 2, left: dims.width / 2} } } opened={ this.state.popover3 } onToggle={ Popeye.toggleHandler(this, "popover3") }> <div className="dropdown"> Aye aye, Captain! </div> </Popeye> </a>

Functions as Offsets

You can also use functions for the offsets. Popeye passes the anchor and itself (in the form of domNodes) to the function.

You must return an object of the same format as the non function version of the offsets.

click me 3
<a key="anchor3" className="button button-main" onClick={ Popeye.toggleHandler(this, "popover3") }> click me 3 <Popeye anchorOffset={ (anchor, pop) => { const dims = anchor.getBoundingClientRect(); return { top: dims.height / 2, left: dims.width / 2} } } opened={ this.state.popover3 } onToggle={ Popeye.toggleHandler(this, "popover3") }> <div className="dropdown"> Aye aye, Captain! </div> </Popeye> </a>

Altogether Now

Now we apply the two to mimic Google's Material Design guidelines on popovers.

The MDG advises us to make the popover cover the button from which it emanates.

Both anchorOffset and popOffset have the same value because we would like them both to be aligned to the top-left. The numerical values we assign them only control the origin of the animationâ€”the center point of the anchor.
